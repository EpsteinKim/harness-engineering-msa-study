# Core Beliefs

> 이 프로젝트의 기술적 신념. 원칙(CONSTITUTION.md)보다 더 근본적인, 팀의 가치관을 기술한다.

---

## 1. 대기열은 단순해야 한다

대기열 시스템의 핵심 가치는 **공정성**과 **신뢰성**이다.
복잡한 우선순위 로직보다 "먼저 온 사람이 먼저 처리된다"는 FIFO 원칙을 기본으로 한다.

## 2. 실패는 투명하게

서비스 장애 시 사용자에게 현재 상태를 정직하게 알린다.
"잠시만 기다려주세요"보다 "현재 대기 인원: N명, 예상 대기 시간: M분"이 낫다.

## 3. 실습을 통한 학습

이 프로젝트는 MSA 실습 프로젝트이다.
프로덕션 완성도보다 **각 기술 요소를 직접 경험하고 이해하는 것**이 우선이다.
단, 학습이라는 이유로 원칙을 무시하지는 않는다.

## 4. 점진적 복잡도

Docker 단일 컨테이너 → Docker Compose MSA → Kubernetes 순으로 진행한다.
각 단계에서 충분히 이해한 후 다음 단계로 넘어간다.
