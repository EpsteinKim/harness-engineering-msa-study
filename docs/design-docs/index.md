# Design Docs (ADR) Index

> 아키텍처 결정 기록(Architecture Decision Records).
> 중요한 기술적 결정의 배경, 선택지, 결정 근거를 기록한다.

## 작성 규칙

- 파일명: `ADR-{번호}-{주제}.md` (예: `ADR-001-service-boundary.md`)
- 한번 작성된 ADR은 수정하지 않고, 대체 ADR을 새로 작성한다.
- 상태: `proposed` → `accepted` / `rejected` / `superseded`

## ADR 템플릿

```markdown
# ADR-{번호}: {제목}

- **상태**: proposed | accepted | rejected | superseded by ADR-{번호}
- **날짜**: YYYY-MM-DD
- **결정자**: {이름/역할}

## 맥락
{어떤 문제를 해결해야 하는가?}

## 선택지
1. {선택지 A} - 장점/단점
2. {선택지 B} - 장점/단점

## 결정
{무엇을 선택했고, 왜?}

## 결과
{이 결정으로 인해 예상되는 영향}
```
